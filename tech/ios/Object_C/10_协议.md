# 协议


- 相当于 Java 中的 Interface

```obj-c
@protocol IBuyTicket <NSObject>
- (void) buyTicket: (NSString *) filmName;
@end
```


Protocol中声明的方法可以实现也可以不实现，不实现只会发出警告，而不会报错。
OC中类不支持多继承，但protocol支持多继承。
Protocol一般定义在.h中。
协议默认必须NSObject的协议。

- 定义协议


协议也是两个类之间的接口定义。
定义协议的类可以看作将协议定义的方法代理给了实现它们的类。

```
// 代理
@protocol 协议名称 <其他协议名称1，其他协议名称2，其他协议名称3>
//方法声明列表
@end
```

- 使用协议
```
// 代理类
@interface 类名：父类 <协议名称1，协议名称2，协议名称3>
@end
```

- protocol中的@required和@optional 
@required和@optional : 方法是否必须实现。
@required —— 必须实现（编译器默认是@required，若不实现，会警告）
@option —— 可以选择实现，用于提供代码提示。

- 非正式(informal)协议？
它实际上是一个分类，列出了一组方法但并没有实现它们。
非正式协议通常是根类定义的，每个人（或几乎每个人）都继承相同的根对象。
有时非正式协议被称为抽象协议。

`@optional`指令在Objective-C 2.0 中，用于取代非正式协议的使用。
